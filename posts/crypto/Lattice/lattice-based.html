                                            # 再探格密码

                                            关于格密码的学习从很久以前已经开始，本篇博文主要是从头梳理再做一次系统的记录。

                                            # 前言

                                            # 什么是格

                                            我们可以这样定义格，它是一组向量在nnn维空间中有着周期性的点集，或者说对于nnn个线性无关的向量v1,v2,…,vn∈Rnv_1,v_2,\dots,v_n \in \mathbb{R}^nv1​,v2​,…,vn​∈Rn，这些向量的整系数组合构成的点集即为一个格，即

                                            L(v1,v2,…,vn):={∑i=1nαivi∣αi∈Z} L(v_1, v_2, \dots, v_n) := \left\{\sum_{i=1}^n{\alpha_iv_i} \Big| \alpha_i\in\mathbb{Z} \right\} L(v1​,v2​,…,vn​):={i=1∑n​αi​vi​​αi​∈Z}

                                            我们把v1,v2,…,vnv_1, v_2, \dots, v_nv1​,v2​,…,vn​称为格LLL的一组基。

                                            从18世纪开始，拉格朗日、高斯和闵可夫斯基等数学家都对格有研究。不过我们主要是看格在密码学中的应用，1996年Ajtai发现格不仅可以用来分析密码还可以来构建密码，同时发现用格来构建密码体系的安全性基于格问题的最坏情况。所以只要解决了某一格密码体系，就可以解决基于此类格问题构建的任何密码（其他密码体系都是基于平均性假设）。

                                            上面这句话的意思就是在格密码体系中我们可以避免很多特殊要求，例如在RSA中，我们构建NNN时要避免选择一些特殊的因子（因为可能会让分解很容易，关于特殊因子导致的RSA分解例子太多了），所以此时我们相当于是一个黑名单机制，不断地选择因子，不断地发现一些因子“不好”，从而避免选择它们。而在格密码中，我们无须考虑这类问题，因为攻击同规格的格问题难度几乎一致。

                                            同时格密码还有下列优势

                                            1. 涉及的计算简单，因为都是有限域运算。
                                            2. 抗量子攻击，这很重要。

                                            # 格问题

                                            SVP（最短向量问题）是格中最主要的问题。其内容为给定一个格基，找出格中的最短非零向量，在高维格中这是一个困难问题，所以我们更多关注它的变体，例如

                                            1. apprSVP（近似最短向量问题），找到近似最短非零向量sis_isi​，满足∥si∥≤γ∥u∥\lVert s_i\rVert\le \gamma\lVert u\rVert∥si​∥≤γ∥u∥，其中uuu代表格中其他任意向量。
                                            2. SIVP（最短线性无关向量问题），找到nnn个线性无关向量sis_isi​，满足∥si∥≤Λn(L)\lVert s_i\rVert\le \Lambda_n(L)∥si​∥≤Λn​(L)
                                            3. uSVP（唯一最短向量问题），找到最短非零向量sis_isi​，满足它比所有其他非平行最短向量短。
                                            4. CVP（最近向量问题），给定目标向量t∈Rnt\in \mathbb{R}^nt∈Rn，找到格中非零向量sis_isi​，满足∥si−t∥≤∥sj−t∥\lVert s_i -t\rVert \le \lVert s_j -t\rVert∥si​−t∥≤∥sj​−t∥，其中sjs_jsj​代表格中其他非零向量。

                                            SVP的难度来自于格中有不同的基，现在我们经常使用的LLL算法难度近似为2O(n)2^{O(n)}2O(n)，不过它在应用中总是很有效，只能说可能是LLL的界太松了，后续还有很多其他优化后的其他算法，但对于要求SVP的解还是需要2O(n)2^{O(n)}2O(n)的运算时间。

                                            对于apprSVP，一般我们认为ncn^cnc-apprSVP当c≥12c\ge\frac{1}{2}c≥21​时非常困难，这里的ncn^cnc为多项式因子，代表随着n的增大，算法所需时间将呈多项式增长。

                                            # 格应用

                                            # 哈希函数

                                            Ajtai最初提出了一类单向函数 [2]，安全性基于ncn^cnc-apprSVP的最坏难度，后续的工作都是来减少常数ccc来提高安全上限。

                                            上述内容都是利用格来构造哈希函数提高抗碰撞能力，因为这不是我的研究方向，故在此不做深究。

                                            # 公钥密码

                                            Ajtai在上述的基础上又和Dwork构建了一个公钥密码体系，其安全性基于uSVP的最坏难度，这里我们只看一下改进后的该密码系统 [3]

                                            给定大整数NNN，选取私钥h∈[N,2N]h\in[\sqrt{N}, 2\sqrt{N}]h∈[N​,2N​]，公钥为从0,1,…,N−10,1,\dots,N-10,1,…,N−1中选取mmm（m=O(log⁡N)m=O(\log N)m=O(logN)）个数a1,…,ama_1,\dots,a_ma1​,…,am​，并且要求他们满足都接近N/hN/hN/h的整数倍，并且包含一个索引i0∈[m]i_0\in [m]i0​∈[m]要求ai0a_{i_0}ai0​​接近N/hN/hN/h的奇数倍。

                                            加密时进行按位加密，对于比特0，将{a1,…,am}\{a_1,\dots,a_m\}{a1​,…,am​}的随机子集求和并模NNN，对于比特1，则在求和时加上⌊ai0/2⌋\lfloor a_{i_0}/2\rfloor⌊ai0​​/2⌋。

                                            解密时则将www模N/hN/hN/h，如果余数很小则为0，否则为1。

                                            正确性证明：a1,…,ama_1,\dots,a_ma1​,…,am​都接近N/hN/hN/h整数倍，所以任意子集之和也接近N/hN/hN/h整数倍，所以对于0的加密结果模N/hN/hN/h的余数很小，同理⌊ai0/2⌋\lfloor a_{i_0}/2\rfloor⌊ai0​​/2⌋不接近N/hN/hN/h整数倍，所以1的加密结果模N/hN/hN/h余数很大。

                                            安全性证明涉及到上述抗碰撞哈希函数的安全性证明，大致就是像格中添加足够的高斯噪音，得到一个接近均匀分布的分布，再从这个分布的统计距离来分析格问题的困难度，详细可参照 [1] 中第六节内容。

                                            很明显上述密码体系相对于如今的格密码体系效率非常之低，用nnn维的格就需要O(n4)O(n^4)O(n4)大小的公钥，且每个加密比特都会变成O(n2)O(n^2)O(n2)大小，显然不符合如今的加密系统需求。

                                            后续Regev提出了基于格基的LWE问题密码体系 [4]，关于LWE不会在本篇做过多介绍，后面可能我会写单独的博文，它基于SVP的最坏情况量子难度，也就是说破解该密码就能有高效的量子算法攻击apprSVP，不过注意上述两个密码系统安全性无法比较优劣，一是第一个基于uSVP而不是SVP，二是LWE显示出SVP可能有量子算法。不过目前还没有量子算法可以超越经典算法解决格问题，所以我们依然可以任务在量子情况下格问题也是困难的。

                                            # 格的性质

                                            上面我们介绍了一些关于格的基本定义和历史工作，从这里开始我们将系统性学习格相关理论和性质。

                                            # 格定义

                                            在群论中，我们通常将格定义为Rn\mathbb{R}^nRn的离散子群，它们的具体定义如下：

                                            离散性：设DDD是Rn\mathbb{R}^nRn的一个子集，当它没有聚点时，即任意x∈Dx\in Dx∈D，都存在r>0r>0r>0使得B(x,r)∩D={x}\mathcal{B}(x, r)\cap D = \{x\}B(x,r)∩D={x}，其中B(x,r)={y∈Rn:∥x−y∥<r}\mathcal{B}(x, r) = \{y\in \mathbb{R}^n: \lVert x - y\rVert < r\}B(x,r)={y∈Rn:∥x−y∥<r}，简单点说就是以该点为圆心半径为rrr的球内没有其他点。

                                            格：(Rn,+)(\mathbb{R}^n, +)(Rn,+)的加法子群，如果它具有离散性则称为格

                                            关于格结构的抽象定义我们只了解到此，我们更关注计算上的内容。在前面我们已经描述了使用线性无关的向量组作为一组基来表示或者说生成格，对于基向量构成的矩阵BBB成为格的格基，则格的表示可以写为

                                            L(B):={Bx:x∈Zn} L(B) := \{Bx : x\in\mathbb{Z}^n\} L(B):={Bx:x∈Zn}

                                            接下来我们来考虑格的秩，格的秩ddd即子空间span(L)\mathrm{span}(L)span(L)的维度，注意格并不一定是满秩的，因为并不要求这组线性无关向量的个数是nnn，当且仅当d=nd=nd=n时格才是满秩的。

                                            还要注意的是，格基一定是构成子空间的一个极大无关向量组，但子空间的每个极大无关向量组不一定能构成格基。对于Rn\mathbb{R}^nRn中的秩为ddd的格，如果B=(b1,…,bd)B=(b_1,\dots,b_d)B=(b1​,…,bd​)是ddd个无关向量，当且仅当不存在非零格向量x∈Lx\in Lx∈L使得x∈P(B)x\in P(B)x∈P(B)时BBB为LLL的一组基。其中P(B)P(B)P(B)代表一组向量生成的最小线性子空间（parallelepiped span），或者说n维平行空间。

                                            P(B)={∑i=1mλibi:∣λi∣<1} P(B) = \left\{\sum_{i=1}^m{\lambda_i b_i} : |\lambda_i|<1\right\} P(B)={i=1∑m​λi​bi​:∣λi​∣<1}

                                            充分必要证明：因为BBB是格基，则每个向量都可以由BBB的整系数组合构成，也就是x=∑i=1dλibi, λi∈Zx = \sum_{i=1}^d{\lambda_i b_i},\ \lambda_i\in\mathbb{Z}x=∑i=1d​λi​bi​, λi​∈Z，如果x∈P(B)x\in P(B)x∈P(B)，则∣λi∣<1|\lambda_i|<1∣λi​∣<1，此时只有格点即零向量在P(b)P(b)P(b)中。反之如果BBB不是格基，则格中存在非零向量x∈P(B)x\in P(B)x∈P(B)，因为此时xxx不是BBB的整系数线性组合，又因为BBB也是LLL中ddd个无关向量，span(L)=span(B)\mathrm{span}(L) = \mathrm{span}(B)span(L)=span(B)，所以xxx能被BBB表示（但系数不全是整数）,记

                                            x=∑i=1dλibi x = \sum_{i=1}^d{\lambda_i b_i} x=i=1∑d​λi​bi​

                                            同时我们考虑对λi\lambda_iλi​取整，记

                                            x′=∑i=1d⌊λi⌋bi x^{'} = \sum_{i=1}^d{\lfloor\lambda_i\rfloor b_i} x′=i=1∑d​⌊λi​⌋bi​

                                            # 模体积（Volume）

                                            对于秩为ddd的格LLL，一组格基为B=(b1,…,bd)B = (b_1,\dots,b_d)B=(b1​,…,bd​)，则

                                            vol(L)=Δ(b1,…,bd)1/2=det⁡(BTB) \mathrm{vol}(L) = \Delta(b_1, \dots, b_d)^{1/2} = \sqrt{\det(B^TB)} vol(L)=Δ(b1​,…,bd​)1/2=det(BTB)​

                                            简单点说，格的模体积就是格基矩阵行列式的绝对值。对于格中的不同格基，都可以通过乘上一个行列式为±1\pm 1±1的单位矩阵来进行变换，所以同一个格无论取哪一组格基，其vol(L)\mathrm{vol}(L)vol(L)都不变，反过来说属于不同格的格基其vol(L)\mathrm{vol}(L)vol(L)不相同。

                                            还有另一种定义模体积的方式，对于Rn\mathbb{R}^nRn中的满秩格LLL，有r>0r>0r>0

                                            vol(L)=lim⁡r→∞Vr∣B(0,r)∩L∣ \mathrm{vol}(L) = \lim_{r\rightarrow\infty}{\frac{V_{r}}{|\mathcal{B}(0, r)\cap L|}} vol(L)=r→∞lim​∣B(0,r)∩L∣Vr​​

                                            这里就是说记以格点为中心半径为rrr的nnn维球B(0,r)\mathcal{B}(0, r)B(0,r)中的格点数为N(r)N(r)N(r)，nnn维球的体积是VrV_{r}Vr​，根据测度理论，rrr足够大时，B(0,r)\mathcal{B}(0, r)B(0,r)中的格点数近似等于其中的向量数，因此有上述模体积的定义。

                                            # 派生格

                                            • 子格

                                              如果格L∈RnL\in\mathbb{R}^nL∈Rn的子集M⊂LM\subset LM⊂L且MMM也是格，则MMM称为LLL的子格。每个子格都是子群，同时每个具有离散性的子群也都是子格，当dim⁡(M)=dim⁡(L)\dim(M)=\dim(L)dim(M)=dim(L)是称为全秩子格（full-rank）。

                                              不同于子空间，子格和LLL能有相同的秩但不等于LLL，此时考虑商集L/ML/ML/M，其中的元素个数我们称为群指数（group index），记为[L:M][L: M][L:M]，对于全秩子格MMM来说，有

                                              vol(M)=vol(L)[L:M] \mathrm{vol}(M) = \mathrm{vol}(L)[L: M] vol(M)=vol(L)[L:M]

                                              不是全秩子格的秩严格小于ddd，所以群指数无限大，则没有上述关系。

                                              还有一个重要的子格关系为原始子格（primitive sublattice），对于秩为ddd的格LLL，当且仅当秩为kkk的子格MMM的基BBB可以通过添加d−kd-kd−k个向量扩充为原格的一个基，则称MMM为原始子格。

                                            • 对偶格（Dual lattice）

                                              一个格的每个元素都能视作另一个格的元素，对于格LLL，对偶格L×L^{\times}L×为

                                              L×={y∈span(L):⟨x,y⟩∈Z ∣ x∈L} L^{\times} = \{y\in \mathrm{span}(L) : \langle x,y\rangle\in\mathbb{Z}\ |\ x\in L\} L×={y∈span(L):⟨x,y⟩∈Z ∣ x∈L}

                                            # 格基归约

                                            # NTRU

                                            关于NTRU的详细介绍可见 [14]，它是一种安全性基于多项式环上的SVP构造的公钥密码体系，目前也是后量子密码的热点算法，因为其相比其他后量子公钥密码速度很快。

                                            这里直接引用的是 [15]中对NTRU介绍的例子，生成了一个一维NTRU公私钥。显然可以构造格

                                            L=(1h0q) L = \begin{pmatrix} 1&h\\ 0&q \end{pmatrix} L=(10​hq​)

                                            则有格点v=(f,g)v = (f, g)v=(f,g)，模长为f2+g2<q\sqrt{f^2+g^2} < \sqrt{q}f2+g2​<q​，显然这里的最短向量期望为

                                            σ(L)=vol(L)1/2=q \sigma(L) = \mathrm{vol}(L)^{1/2} = \sqrt{q} σ(L)=vol(L)1/2=q​

                                            所以我们可以认为vvv就是SVP的解或近似解，从而可以通过LLL得到私钥。不过实际上我们在攻击时目标可能并不完全符合密码标准或者说是一类自定义的密码体系，例如对于这个例子，如果我们让fff的值变得更大，就会导致∥v∥>σ(L)\lVert v\rVert > \sigma(L)∥v∥>σ(L)，从而不一定能够从LLL得到解，所以很多时候在实际攻击时我们需要对构造的格进行配平。

                                            (f,−k)L=(f,g) (f, -k)L = (f, g) (f,−k)L=(f,g)

                                            则我们变成

                                            (f,−k)(1Dh0Dq)=(f,Dg) (f, -k)\begin{pmatrix} 1&Dh\\ 0&Dq \end{pmatrix} = (f, Dg) (f,−k)(10​DhDq​)=(f,Dg)

                                            此时我们的目标向量变成(f,Dg)(f, Dg)(f,Dg)，而DDD的取值和fff与ggg的插值有关，一般来说我们的目的就是让向量在每个方向的长度都一致，即变得“更平”。因为对于最短向量来说，某列乘上一个数对模长的影响是累加的，而对格模体积的影响是累乘的，这样我们就可以在原做法的基础上得到一个更大的解的上界。
